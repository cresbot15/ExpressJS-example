# Pros
 - Relatively quick and easy to set up end points
 - Same language front end would be written in
    - Likely leading to fewer mistakes and time spent reading extra documentation
 - Pretty easy to deploy
 - Javascript ecosystem is massive and extremely well supported
 - Can use TS instead of JS if we need static typing
# Cons
- Lacks convention and opinions
    - We have to figure out the structure and design mostly by ourselves (can also be a good thing)
- Must manually handle all possible routes, no automatically returning 5xx codes if an internal error is generated
- Callback syntax can be messy and difficult to read
- Promises can be extremely confusing if we need to implement async code (we most likely will)
    - Syntax can also get extremely messy
- No inbuilt autogenerated documentation

# Final Thoughts
Being able to write our back end in the same language as our front end is a slight upside. Javascript's ecosystem is also very old and extremely well developed so finding answers to our questions won't be difficult. There are is also an NPM package to solve just about any problem. Typescript can alleviate some of Javascript's issues.

Writing asynchronous code in Javascript can be big hassle as it's very easy to write code that's almost impossible to understand and maintain.
